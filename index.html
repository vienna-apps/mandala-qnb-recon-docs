III. Recon Process

The reconciliation compares records from Bersama Activity Report (source NEW_BERSAMA) against Report Equation Network (source ATMB_EQUATION) using a computed reconId on the new_bersama_recon table.

Field Cross-Reference

FieldBersama .rptEquation .xlsRecon Key?

Transaction DatetrDate (pos 0–8)satstp (date part)
Amountamount (pos 137–154)saama
Card NumbercardNumber (pos 80–100)— (embedded in sgnr1)
Terminal IDtrmlId (pos 52–70)—
Receipt NumberreceiptNbr (pos 201–215)—

Match Algorithm

  • For each Bersama record, compute SHA1 reconId based on transaction type
  • For each Equation record, compute SHA1 reconId from corresponding SAP fields
  • Link records in new_bersama_recon via reconId — storing bersamaIssId, bersamaBnfId, bersamaAcqId, and equationNetworkId
  • Records with matching reconId → status MATCH; otherwise UNMATCH
  • Settlement totals written to recon_settlement table for downstream Network recon

SFTP / FTP Config

There are three mechanisms for fetching files from SFTP/local paths, using two separate config tables:

1. Auto-fetch for Custom Recon (Nostro types) — UploadFileCustomReconController

When a user manually uploads a file for a Custom Recon type, autoFetchMissingConfiguredRoles() is triggered automatically. It queries sftp_config by recon_code + file_role and fetches any remaining required file roles (e.g. DWH balance files) without user interaction. fetchAndUploadConfiguredRoles() is also called at upload submission time.

ColumnPurpose
idPrimary key — for custom recon this is a generated UUID; for standard recon this matches a SftpConfigKey enum value
host"LOCAL" → read from local_directory on the app server; any other value → SFTP hostname

enabledMaster on/off switch for this config row
sftp_config_recon (join table)Additional recon codes sharing the same file source — allows one SFTP config to serve multiple templates

Source type is derived at runtime: SftpConfig.SourceType.fromHost(host) — if host == "LOCAL" → reads local filesystem; otherwise → connects via SFTP (Apache SSHD / JSch).

2. Optional transaction file path for standard recon types — TransactionFileService

For GPN, Tarik Tunai, Transfer ATM, and Pembelian Pulsa recon types, the upload controller calls TransactionFileService.resolveTransactionFile(filename, SftpConfigKey.XYZ_OPT_PATH) to locate an optional counterpart transaction file on the server filesystem. The path and file pattern are read from sftp_config using the SftpConfigKey enum as the row ID.

3. DWH/counterparty auto-fetch for standard recon types — keyed by recon_type

These controllers query sftp_config by recon_type column (matching the ReconType enum name) and fetch files from local path or SFTP. Unlike the Custom Recon mechanism, this is triggered by a user clicking "Fetch from SFTP" on the upload page, not automatically.

4. Legacy SFTP fetch — ftp_upload table via SftpService

An older mechanism using a separate ftp_upload config table (not sftp_config). The controller calls SftpService.connectToSftp(dateString, id) where id is the primary key in the ftp_upload table. Triggered by a "Fetch from SFTP" or "Get File from FTP" button on the upload page.
